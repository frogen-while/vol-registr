# Техническое задание: Match-страница

## 1. Контекст и цель

Проект: Pocket Aces Tournament Platform.

Цель Match-страницы: универсальный публичный хаб матча и этапа турнира.

Ключевой акцент:
- Расписание по этапам (доминантный блок после Hero)
- Детальная пост-матч статистика игроков
- Dream Team по позициям (авторасчет)
- MVP матча и MVP турнира
- Архив матчей внутри Match-раздела

Важно: прямая трансляция не гарантирована, поэтому система должна полноценно работать в режиме без стрима.

## 2. Аудитория

- Смешанная аудитория: болельщики, игроки, организаторы
- Языки интерфейса: EN, PL

## 3. Формат турнира

- Однодневный турнир
- 12 команд
- Этапы: Группы -> Плей-офф -> Финал
- Возможны 3 параллельных матча

## 4. Роли и права

- Публичный пользователь:
  - Просмотр расписания, статистики, рейтингов, страниц игроков и команд
  - Экспорт PDF и PNG social-card
- Админ/организатор:
  - Импорт CSV после матча
  - Повторный импорт с полной перезаписью
- Супер-админ:
  - Ручная правка MVP, Dream Team, Highlights

Примечание: механизм публичных оспариваний статистики не требуется.

## 5. Источник данных и режим публикации

Источник статистики: SoloStats Live, без live API.

Следствия:
- Основной режим страницы: post-match
- Импорт: вручную через CSV
- SLA публикации: до 30 минут после матча
- Если CSV не загружен или импорт упал: показывать только счет и этап
- Если данные неполные: публиковать частично с явными пометками пропусков

## 6. Структура Match-страницы

### 6.1 Hero-блок

Обязательно:
- Счет по сетам
- Текущий статус матча

Опционально (если стрим есть):
- Встроенный плеер прямо в Hero под счетом

Если стрима нет:
- Заглушка
- Иконка статуса стрима (без текстового статуса)
- Fallback-блок: расписание ближайших матчей

Если стрим недоступен в моменте:
- Сообщение о недоступности
- Кнопка обновления плеера

### 6.2 Блоки ниже Hero

Порядок приоритета:
1. Расписание этапа (доминантный)
2. Детальная статистика игроков
3. Dream Team по позициям
4. MVP блок
5. Highlights (топ-10)
6. Медиа-галерея (фото + видео)
7. Вкладка архива матчей

## 7. Расписание и навигация по матчам

- Отображение расписания по этапам
- 24-часовой формат времени
- Явная метка таймзоны: Europe/Warsaw
- Переключение параллельных матчей: табы Match 1, Match 2, Match 3
- Автопереход на следующий матч не нужен

## 8. Статистика игроков

### 8.1 Обязательные метрики

- Kills
- Attack Efficiency
- Aces
- Blocks
- Errors
- Reception/Digs
- Assists
- MVP/рейтинг

### 8.2 Фильтры

- По команде
- По позиции
- По этапу
- По конкретному матчу

### 8.3 Таблицы

- Сортировка по всем ключевым колонкам
- Для больших объемов: бесконечный скролл
- На мобильных: компактные карточки игроков

## 9. Dream Team и рейтинги

- Dream Team формируется автоматически
- Топ-3 по каждой позиции
- Позиции:
  - Setter
  - Opposite
  - Outside Hitter
  - Middle Blocker
  - Libero
- Алгоритм: отдельная формула для каждой позиции
- Дополнительно: отдельные рейтинги по каждой позиции на весь турнир

## 10. MVP и highlights

- Отдельные блоки:
  - MVP текущего матча
  - MVP турнира
- Highlights: топ-10 ключевых моментов
- Ручная правка этих блоков: только супер-админ

## 11. Медиа

- Медиа-блок после матча: фото + видео
- Хранение медиа: внешний CDN/облако

## 12. Публичные страницы и SEO

Должны быть индексируемы:
- Match-страница
- Страница игрока
- Страница команды

Кликабельные переходы:
- Команда -> страница команды
- Игрок -> страница игрока
- Матч -> детальная страница матча

## 13. Экспорт

Публичные форматы выгрузки:
- PDF
- PNG social-card

## 14. UX, a11y, motion, performance

- Уведомления о новых данных: не нужны
- Запоминание фильтров между визитами: не нужно
- Уровень доступности: базовый (контраст + клавиатурная навигация)
- Анимации: максимальные, спортивный стиль
- Обязательно учитывать prefers-reduced-motion
- Допустимое время первичной загрузки на 4G: до 5 секунд
- Приоритет визуала выше скорости, но без критических деградаций

## 15. Ограничения и принятые решения

- Стрим не является обязательным для релиза
- Live-чат не нужен
- Head-to-head блок не нужен
- Сравнение игроков не нужно
- Push/reminder уведомления не нужны
- Audit trail не требуется в рамках согласованного брифа

## 16. Требования к CSV-импорту

- Обязательный предпросмотр перед публикацией
- Валидация формата и отображение ошибок
- Повторный импорт того же матча: полная перезапись данных
- При неполном CSV: публиковать частично с пометками пропусков

## 17. V1 (must-have)

- Расписание по этапам
- Пост-матч статистика игроков из CSV
- Dream Team + MVP
- Отдельная страница игрока
- Фото/видео блок
- Опциональный встроенный стрим (если доступен)

## 18. Roadmap реализации

### Этап 1. Данные и backend

- Спроектировать серверные модели для матчей, этапов, статистики, MVP, highlights
- Реализовать импорт CSV с предпросмотром, валидацией, перезаписью
- Подготовить API для:
  - расписания по этапам
  - статистики игроков
  - Dream Team и позиционных рейтингов
  - MVP и highlights
  - медиа-блока

### Этап 2. Публичный UI Match

- Hero (счет/статус)
- Табы для 3 параллельных матчей
- Доминантный блок расписания этапов
- Статистика игроков с фильтрами, сортировкой, infinite scroll
- Мобильные карточки игроков

### Этап 3. Страницы сущностей

- Детальная страница матча
- Детальная страница игрока
- Переходы между матчем, игроком, командой
- Архив матчей во вкладке Match

### Этап 4. Рейтинги и контент

- Dream Team (авто)
- Позиционные рейтинги турнира
- MVP match/tournament
- Highlights топ-10
- Медиа-галерея

### Этап 5. Стрим и fallback

- Опциональный встроенный плеер в Hero
- Единый fallback для отсутствующего/недоступного стрима
- Поведение без стрима как основной стабильный режим

### Этап 6. Качество и релиз

- Проверка EN/PL локализации
- Проверка индексации и метаданных SEO
- Проверка a11y-базы и reduced-motion
- Профилирование загрузки и оптимизация до целевых 5 секунд

## 19. Критерии приемки

Функциональные:
- Пользователь видит расписание по этапам и может переключать параллельные матчи
- После импорта CSV доступна детальная статистика игроков
- Работают фильтры и сортировка
- Публикуются Dream Team, MVP, highlights
- Работают страницы игрока, команды, матча

Нефункциональные:
- EN/PL доступны
- 24-часовой формат времени + Europe/Warsaw
- A11y базового уровня соблюден
- prefers-reduced-motion поддержан
- При отсутствии стрима страница полностью функциональна
