# Match Page: Sprint 1 Backlog

## Цель спринта

Собрать рабочую V1 Match-раздела в post-match режиме с CSV-импортом, расписанием по этапам, детальной статистикой игроков и базовыми публичными страницами.

## Границы Sprint 1

Входит:
- Расписание по этапам
- Импорт CSV с предпросмотром и валидацией
- Публикация статистики игроков после импорта
- Dream Team (авто), MVP матча и MVP турнира
- Страницы: Match detail, Player detail
- EN/PL локализация для новых экранов
- Fallback без стрима (основной путь)

Не входит (перенос в Sprint 2+):
- Полноценный встроенный live-плеер
- Расширенная медиа-галерея с тяжелыми видео-сценариями
- Тонкая SEO-оптимизация с расширенной схемой

## План задач

## A. Backend

### A1. Модели матчевого контура
Priority: P0

Задачи:
- Добавить/актуализировать модели:
  - Stage
  - Match
  - MatchStatSnapshot (или аналог хранения агрегатов)
  - PlayerMatchStats
  - DreamTeamEntry
  - MVPSelection
  - MatchHighlight
- Связать Match с Team и этапом турнира
- Добавить поля статуса публикации и времени обновления

Definition of Done:
- Миграции создаются и применяются локально без конфликтов
- Админка Django позволяет просматривать основные сущности

### A2. CSV Import Pipeline
Priority: P0

Задачи:
- Реализовать загрузку CSV через admin API
- Реализовать предпросмотр (без коммита)
- Реализовать валидацию формата и обязательных полей
- Реализовать режим полной перезаписи по match_id
- Реализовать частичную публикацию с метками пропусков

Definition of Done:
- Есть endpoint preview и endpoint confirm import
- Повторный импорт корректно перезаписывает статистику матча
- Ошибки формата возвращаются в читаемом виде

### A3. Public API для Match-страниц
Priority: P0

Задачи:
- Endpoint расписания по этапам
- Endpoint матча (детали + счет + статус + ключевые блоки)
- Endpoint статистики игроков с фильтрами
- Endpoint Dream Team и позиционных рейтингов
- Endpoint MVP и highlights

Definition of Done:
- Фильтры работают: team, position, stage, match
- Сортировка поддержана для ключевых колонок
- Формат времени 24h и timezone Europe/Warsaw доступны в ответах

### A4. SEO-ready routing data
Priority: P1

Задачи:
- Подготовить slug/id роуты для публичных страниц:
  - match detail
  - player detail
  - team detail (реюз)
- Добавить поля/методы для canonical URL

Definition of Done:
- Страницы доступны по стабильным URL
- Данные для title/description выдаются сервером

## Риски спринта

- Разнородные CSV-файлы из источника
- Расхождения в позициях/идентификаторах игроков
- Объемы статистики могут перегружать клиент
- Стрим функционально вне критического пути, но может отвлечь сроки

## Антирисковые меры

- Сначала фиксировать контракт CSV-шаблона
- Делать preview-only валидацию до подтверждения импорта
- Добавить defensive fallback UI для любого пустого блока
- Отложить live-плеер за feature flag

## Результат Sprint 1

К концу спринта пользователь должен получить:
- Рабочую Match-страницу с расписанием по этапам
- Публичную пост-матч статистику игроков
- Dream Team/MVP/Highlights
- Навигацию на детальные страницы матча и игрока
- Стабильную работу без обязательной трансляции
